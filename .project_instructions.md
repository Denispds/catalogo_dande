# Dande Acessórios - Catálogo Digital

## Overview
Mobile-first digital catalog for a Brazilian jewelry wholesaler.
- Language: Portuguese (pt-BR) — all user-facing UI and responses
- Tech: Next.js 14.2.28, Prisma 6.7.0, PostgreSQL, S3 (us-west-2), NextAuth
- ~841 products, 8 collections

## Deployment
- Hostnames: `catalogo-dande-11z5g8.abacusai.app`, `dandecatalogo.abacusai.app`, `catalogo-dande.abacusai.app`
- All untagged — single `deploy_nextjs_project` call updates all 3.

## Constraints
- **`next.config.js` is UNEDITABLE** (system-locked). It has `images: { unoptimized: true }` — this means Next.js Image optimization is globally disabled.
- Package manager: yarn only.
- S3 bucket: `abacusai-apps-c7b7903a3d7c9d0629cf50b7-us-west-2` in `us-west-2`.

## Image Optimization Architecture
Since `next.config.js` is locked, a **custom image proxy** provides on-the-fly resize + format conversion:

### `/api/image` route (`app/api/image/route.ts`)
- Query: `?url=<encoded>&w=<32-2048>&q=<10-100>&f=webp|avif|jpeg`
- Uses `sharp` (v0.34.5) for resize + format conversion
- LRU cache (200 items max) + `Cache-Control: public, max-age=31536000, immutable`
- Allowed hosts: S3, supabase, unsplash, googleusercontent, cloudfront, digitaloceanspaces, abacusai.app
- Unknown hosts: gracefully redirect (302) to origin
- Default: WebP @ q78

### Helpers (`lib/image-url.ts`)
```ts
getOptimizedImageUrl(url, width=640, quality=78)
getOptimizedSrcSet(url, widths=[320,480,640,960], quality=78)
```

### `OptimizedImage` component (`components/optimized-image.tsx`)
Plain `<img>` with srcset/sizes/loading=lazy — used by product cards.

### Pages using optimized images
- `components/product-card.tsx` — grid, list, single layouts
- `components/image-lightbox.tsx` — main (q88) + thumbnails (q70, 128w)
- `app/_components/catalog-client.tsx` — main grid (priority for first 4), featured carousel
- `app/colecoes/_components/colecoes-client.tsx` — collection grid thumbnails, previews, search results
- `app/colecoes/[slug]/_components/collection-catalog-client.tsx` — collection cover banner

## API Optimization: `/api/produtos`
DB-level pagination (Prisma `skip/take`) for the fast path (normal sorts).
Hybrid two-stage fetch for discount filter or image-date sort:
1. Query only sort fields for all rows
2. Sort + paginate in memory
3. Fetch full data for just the page's codes

## Performance Results
- API response: **242ms → 12ms** (20× faster)
- Image size: **1.7MB PNG → 46KB WebP** (97% reduction)
- First screen bandwidth: ~60MB → ~1MB

## Key Decisions
- Sharp over `next/image` because global `unoptimized: true` blocks Next image optimization
- Plain `<img>` over `<Image>` for same reason
- WebP @ q78 for most contexts, q85-88 for lightbox/zoomed images
- `priority` (eager loading) for first 4 products above-fold
- Image cache is per-instance (Node process); global caching handled by Cache-Control + CDN/browser
